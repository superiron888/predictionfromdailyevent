# Inversion Engine — 逆向思维引擎

> "Invert, always invert." — Charlie Munger (via Jacobi)

## Purpose
Ensure at least one connection comes from the OPPOSITE direction of consensus. Inversion is not contrarianism for its own sake — it's a systematic way to find what others miss.

## 3 Inversion Methods

### Method A: Invert the Subject
```
Forward: Who BENEFITS from [event]?
Inverse: Who is HURT by [event]? Who benefits from the ABSENCE of [event]?

Example: "感冒多了"
  Forward: Who benefits? → Pharma, telemedicine
  Inverse: Who is hurt? → Airlines (cancellations), restaurants (no-shows), gyms (membership freeze)
  → If flu season is severe, these sectors may quietly underperform
  → Short/underweight angle, not just long
```

### Method B: Invert the Chain
```
Forward: Event → obvious conclusion
Inverse: What if the obvious conclusion is WRONG?

Example: "感冒多了"
  Forward: Flu → pharma benefits
  Inverse: What if pharma does NOT benefit? (flu is priced in, revenue share tiny)
  → Force yourself out of the obvious frame
  → Ask: "If pharma is off the table, what IS affected?"
  → Answer: Behavior changes (居家, 外卖, 刷手机) — completely different direction
```

### Method C: Invert Time
```
Forward: What's happening NOW?
Inverse: What will we think looking BACK in a month?

Example: "好多人感冒了"
  Forward now: Everyone buying pharma, worried about health
  Looking back in 1 month: Flu passed, pent-up demand released
  → Post-flu consumption rebound → restaurants, travel, experiential spending
  → The reversion trade is often overlooked during the event
```

## Substitution Chain Analysis

When A is weakened, don't just find "A's beneficiary." Find who provides A's **FUNCTION**.

```
Flu reduces in-person social interaction
  → Function of social interaction: connection, entertainment, belonging
  → Who provides these functions ONLINE?
    → Social gaming (RBLX), live streaming, virtual communities
  → This is a FUNCTION-level substitution, not a form-level substitution

NOT: "people can't go out → they order delivery" (too obvious)
YES: "people lose social connection → they seek it in virtual spaces" (function-level)
```

**Rule**: The more cross-domain the substitution, the more interesting and non-consensus it is.

## Scale Shift

| Direction | Method | Example |
|-----------|--------|---------|
| Micro → Macro | Personal observation → population trend → macro data → market | "我周围人都感冒" → CDC数据什么水平？→ 若10年最严重 → 宏观劳动参与率影响 |
| Macro → Micro | Big headline → specific industry → concrete verification point | "美联储可能降息" → 对你家楼下小店意味什么？→ 微观验证有时更有投资价值 |

## Mandatory Rule
**At least 1 out of every 2-3 connections MUST come from an inversion method.** This is non-negotiable. Without inversion, output converges to consensus dressed up as insight.
