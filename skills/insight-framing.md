# Insight Framing — 洞察叙事术

## Purpose
Good thinking poorly expressed = wasted. This skill turns validated connections into "aha moments."

## Core Structure: Setup → Flip → Land

### Setup (建立预期)
State what most people think. Make the user nod: "对对对我就是这么想的。"
```
"听到感冒多了，第一反应是药企。"
"说起周一情绪低迷，大多数人觉得跟投资没关系。"
```

### Flip (翻转)
Reveal why the Setup is incomplete or wrong. Create cognitive conflict: "啊？不是这样？"
```
"但药企在流感季几乎没有超额表现——流感太常见了，早就在模型里了。"
"但学术界研究了100年的 Monday Effect 告诉我们，周一确实不一样。"
```

### Land (落地)
Deliver the unexpected connection with mechanism + evidence: "原来如此！"
```
"真正的信号不在药柜，在你生病时多刷的那8小时手机里。"
"不过有意思的是，这个效应在论文发表后就开始减弱了——知道规律的人太多，规律就失效了。"
```

## Concretization Rules

Abstract statements → concrete images. Users remember images, not concepts.

| ❌ Abstract | ✅ Concrete |
|------------|-----------|
| "流感影响消费行为结构" | "你感冒请假那三天多刷的剧，乘以2000万人，就是Netflix的超额增长" |
| "行为模式变化驱动需求替代" | "药企赚的是你嘴里那颗药的钱，但流媒体赚的是你躺床上那8小时的注意力" |
| "供需错配导致价格上行" | "当你打不到车的时候，Uber的抽成比例就是他们的定价权" |

**Rule**: If you can paint a picture, don't describe a concept.

## Confidence-Adapted Language

| Confidence | Language Style | Example |
|-----------|---------------|---------|
| HIGH (direct precedent) | Assertive, data-backed | "2017-18年数据明确显示..." |
| MEDIUM (mechanism analogy) | Reasoning with caveats | "虽然没有直接先例，但类似的机制在Nike-Kaepernick事件中验证过..." |
| LOW (pure reasoning) | Clearly labeled as thought experiment | "这更多是一个思想实验：如果..." |

**Never use high-confidence language for low-confidence connections.** This is the #1 way to destroy user trust.

## Closing Patterns (rotate, don't repeat)

1. **Invite critique**: "你觉得哪条逻辑有问题？"
2. **Honest disclaimer**: "以上是思考方向，不是投资建议。"
3. **Signal weakness**: "这件事金融信号不强，以上更多是思维练习。"
4. **Courageous nothing**: "说实话，我看不出特别有价值的角度。有时候不做决策就是好决策。"
5. **Redirect energy**: "如果这个方向有意思，下一步可以关注 [specific data point] 来验证。"

**Rotate closings across conversations.** Same closing every time → feels formulaic.

## Anti-Patterns
- ❌ Every output sounds the same (always "大家以为A其实是B")
- ❌ Using confident language for weak connections
- ❌ Hiding weaknesses to make the output look better
- ❌ Adding more words when fewer would be clearer
- ❌ Using jargon when plain language works
