# Butterfly Effect Chain — Causal Reasoning Chain Builder

## When to Use
When reasoning from a daily event to financial market connections. This is the primary reasoning structure for daily-life inputs.

## Three Laws

1. **Everything is connected**: Behind any observation lies multiple paths to financial markets.
2. **Shorter chains, higher confidence**: 3 steps > 5 steps > 7 steps. Each step amplifies uncertainty.
3. **Cross-validation boosts confidence**: Same conclusion from 2 different disciplines = stronger signal.

## Chain Construction

### Phase 1: Event Deconstruction
Break user input into atomic events:
```
"好多人感冒了"
├── Direct: Flu/cold prevalence elevated
├── Causes: Virus spread, temperature change, immune suppression
├── Temporal: Current season, recent weather, CDC trends
└── Scale: Individual → Group → Population-level
```

### Phase 2: Multi-Dimensional Chain Generation
For each dimension, build a chain with max 3 links:
```
chain: [Event] → [Mechanism] → [Industry/Theme]
discipline: [Which field's knowledge]
link_strength: strong / weak (for each link)
time_horizon: immediate / 1-3 weeks / 1-3 months
```

### Phase 3: Quality Checks (each chain must pass)
- **Logical coherence**: Each step derives from the previous?
- **Disciplinary accuracy**: Referenced principles correct?
- **Explicit assumptions**: What's assumed? Is it reasonable?
- **Counter-argument**: When would this chain fail?
- **Temporal consistency**: Time scales of each step match?

## Anti-Patterns
1. ❌ Mystical reasoning (astrology, feng shui)
2. ❌ Numerical coincidence ("today is the 8th, so...")
3. ❌ Hindsight bias as sole basis
4. ❌ Emotional contagion ("I feel it will rise")
5. ❌ Infinite extrapolation (short-term → long-term trend)
