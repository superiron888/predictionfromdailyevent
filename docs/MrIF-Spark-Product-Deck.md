## Slide 2 — 我们解决什么问题

### ⚠ 散户投资者的真实困境

| 困境 | 描述 |
|------|------|
| 信息爆炸，角度单一 | 每天被新闻轰炸，但思考方向只有"涨/跌/买/卖"，视野狭窄 |
| 共识即陷阱 | 人人都知道的信息 = 已经 priced in = 没有 alpha |
| 跨学科盲区 | 感冒了想到药企，地震了想到保险——第一反应往往是"正确的废话" |
| 不知道从哪里开始想 | 面对一个事件，不知道如何从 0 到 1 建立金融思考框架 |
| 工具要么太浅要么太深 | ChatGPT 太泛（没有金融逻辑），Bloomberg 太专（看不懂或没权限） |

### ✅ Spark 的定位

| | 传统金融工具 | 通用 AI (ChatGPT) | **Mr.IF Spark** |
|---|---|---|---|
| 定位 | 给你答案 | 给你信息 | **给你角度** |
| 输出 | "买 NFLX" | "流感对经济有影响" | "生病的人不买药，他们买的是无聊的解药" |
| 需要专业知识 | 高 | 低 | **低门槛，高上限** |
| 说"没什么信号" | 不会 | 不会 | **会，而且这是核心价值** |
| 承认可能错 | 很少 | 偶尔 | **每一条都标注弱点** |

**一句话**：Spark 不告诉你买什么，Spark 帮你想到你没想到的。

## Slide 3 — 产品工作原理

### 四步推理流程

```
用户输入（任何日常观察）
        ↓
   ┌─────────────────┐
   │  Step 0: 信号解码  │  signal_decode
   │  提取 → 分类 → 量级  │  12 种事件类型
   │  → 是否需要事实验证    │  micro/medium/macro
   └────────┬────────┘
            ↓
   ┌─────────────────┐
   │  Step 1: 共识映射  │  （内置于推理流程）
   │  大多数人会怎么想？   │  标注认知偏差
   │  这些想法哪些是陷阱？  │
   └────────┬────────┘
            ↓
   ┌─────────────────┐
   │  Step 2: 连接生成  │  insight_engine
   │  6种桥接 × 正/逆    │  12 种可能方向
   │  = 5-6 条候选连接   │  强制≥1条逆向
   └────────┬────────┘
            ↓
   ┌─────────────────┐
   │  Step 3: 对抗验证  │  reality_check
   │  因果测试 → 历史验证  │  4 道关卡
   │  → 做空者攻击 → 共识  │  杀掉弱连接
   │  过滤                │
   └────────┬────────┘
            ↓
   ┌─────────────────┐
   │  Step 4: 叙事输出  │  Setup → Flip → Land
   │  2-3 条存活连接     │  ≤400 字
   │  + 诚实弱点 + 邀请反驳 │
   └─────────────────┘
```

## Slide 4 — 6 种桥接方法

### 从日常事件到金融连接的 6 条桥

| # | 桥接类型 | 正向思考 | 逆向思考 |
|---|---------|---------|---------|
| 1 | **行为桥** (4 类主体) | 事件改变了谁的行为 → 哪个行业受影响？ | 谁因为这个行为变化而受损？ |
| 2 | **资源桥** | 事件改变了什么供需 → 谁依赖它？ | 如果供需变化被对冲，什么替代品上升？ |
| 3 | **叙事桥** | 事件改变了什么情绪 → 情绪驱动什么行为？ | 情绪过度反应时，反转在哪里？ |
| 4 | **替代桥** | A 变弱 → 谁提供 A 的功能（不是形式）？ | A 变强 → 谁的替代价值下降？ |
| 5 | **尺度桥** | 微观观察 → 宏观趋势 → 市场含义 | 宏观标题 → 微观验证点 |
| 6 | **时间桥** | 即时(已定价) → 滞后(1-3周甜蜜点) → 延迟(1-3月) → 反转(3-6月) | 市场忽略了哪个时间窗口？ |

### 4 类行为主体（行为桥的核心）

| 主体 | 行为类型 | 举例 |
|------|---------|------|
| **消费者** | 消费、出行、时间分配、注意力、渠道选择 | 感冒→居家→刷手机→流媒体 DAU↑ |
| **企业** | 招聘、资本支出、库存、风险姿态、战略转向 | 关税→供应链转移→东南亚 capex↑ |
| **机构** | 资产配置、风险限额、板块轮动、合规调整 | 地缘冲突→EM减仓→美国回流 |
| **政策** | 立法、执法、支出、利率决策、应急响应 | 流感大爆发→CDC升级→公卫拨款↑ |

## Slide 5 — 对抗性验证（防 Beardstown Ladies）

### 为什么需要"镜子"

> 1990 年代，Beardstown Ladies Investment Club 声称年化 23%，碾压 S&P 500。
> SEC 审计后发现：她们把追加投入的本金当成了投资收益。
> 实际年化 ~9%，甚至略低于指数。

**教训**：看起来聪明的分析 ≠ 真正有价值的分析。需要内置"攻击者"。

### Spark 的 4 道验证关卡

| Gate | 名称 | 做什么 | 杀掉什么 |
|------|------|--------|---------|
| 1 | **因果测试** | 时间顺序、机制清晰度、排他性、量级合理性 | 逻辑跳跃、夸大效应 |
| 2 | **历史验证** | 找到先例 + **强制找反例** | 只有支持证据、没有反面的连接 |
| 3 | **做空者攻击** | 切换到攻击者角色，不是"polite pre-mortem" | 经不住一个反问就崩塌的连接 |
| 4 | **共识过滤** | 🟢低/🟡中/🔴高 共识标签 | 全部🔴时诚实说"没什么信号" |

### 额外机制

- **强制逆向**：每 2-3 条连接中，至少 1 条必须来自逆向/反常识方向
- **模式单调检查**：检测是否在重复自己的套路（总是用行为桥、总是指向流媒体）
- **置信度透明**：HIGH（直接先例）/ MEDIUM（机制类比）/ LOW（纯推理），语言随之调整

## Slide 6 — 输出格式

### Setup → Flip → Land 叙事结构

| 阶段 | 作用 | 用户心理 |
|------|------|---------|
| **Setup** | 说出大多数人的想法 | "对，显然是这样" |
| **Flip** | 揭示这个想法为什么不完整或错误 | "等等，什么？" |
| **Land** | 给出意想不到的连接 + 机制 + 证据 | "嗯…这个有意思" |

### 每条连接的结构

```
🔗 [挑衅性标题]

[Setup → Flip → Land，3-5 句话，融入历史案例]

📚 历史印证: [年份，事件，什么动了，动了多少]
👀 验证信号: [1-2 个可观察信号]
⚠️ 最脆弱环节: [诚实弱点]
[🟢/🟡/🔴 共识标签]
```

### 输出规则

| 规则 | 说明 |
|------|------|
| 语言 | 用户说中文就中文，金融术语保持英文 |
| 长度 | 每条 ~100-150 字，总计 ≤400 字 |
| 无推荐 | 只到板块/主题级别，ticker 仅作示例 |
| 无概率 | 不说"60% 概率"，用定性描述 |
| 可以说没什么 | "这件事金融信号不强" — 这是核心价值 |
| 邀请反驳 | 每次都以问题或邀请结尾 |

## Slide 7 — 自适应深度

### 不同输入 → 不同深度

| 输入类型 | 示例 | 输出策略 |
|---------|------|---------|
| 🔵 微观/轻松 | "我打了个喷嚏" | 1-2 条轻松连接，可以有趣味。如果真的没信号，说"这更多是思维练习" |
| 🟢 中等事件 | "好多人感冒了" | 2-3 条标准深度，每条有历史案例，≥1 条逆向 |
| 🔴 宏观/复杂 | "俄乌冲突升级" | 2-3 条深度连接，丰富历史上下文，覆盖多传导通道 |
| ⚪ 低信号 | "周一了情绪低迷" | 承认已知模式（如有），给 1 个有趣角度，诚实说信号弱 |
| 🟡 混合信号 | "放假了不想出门" | 拆分信号，聚焦最有趣的那个 |

## Slide 8 — 知识体系

### 12 个 Skill 模块

#### 金融骨架层（7 个，改编自 Mr.IF v4.3）

| Skill | 功能 |
|-------|------|

| cross-domain-reasoning | 多学科金融映射（心理学→社会学→地缘→供应链→化学→市场传导），含量化锚点 |
| second-order-thinking | 二阶思考 5 工具（So What 测试、Priced In 清单、隐形赢家搜索、时间错配、预期方向） |
| historical-precedent | 历史先例评估框架 + 强制反例搜索 |
| event-classification | 12 类事件分类 + 判断逻辑 + 验证决策 |
| novel-event-reasoning | 无模板新事件的第一性原理分析：跟着钱走 |

#### 创意层（5 个，Spark 专属）

| Skill | 功能 |
|-------|------|
| consensus-decode | 共识解码：映射一阶反应，标注认知偏差（可得性、锚定、叙事谬误、幸存者、时间压缩） |
| behavioral-propagation | 行为传播引擎：4 类主体 × 2-3 阶效应 × 时间错配（滞后甜蜜点） |
| inversion-engine | 逆向引擎：3 种逆向方法（主体/链条/时间）+ 替代链分析 |
| anti-confirmation | 对抗性自检：攻击者角色 + 共识过滤 + 模式单调检查 |
| insight-framing | 洞察叙事：Setup→Flip→Land + 具象化规则 + 置信度适配语言 |

## Slide 9 — 3 个核心工具

### MCP 工具链

| 工具 | 输入 | 输出 | 作用 |
|------|------|------|------|
| `signal_decode` | 用户原始输入 | 事件类型 + 量级 + 是否需验证 + 桥接建议 | 解码信号，校准深度 |
| `insight_engine` | 用户输入 + 事件类型 | 链模板 + 共识陷阱 + 逆向提示 + 历史案例 | 引导创意推理 |
| `reality_check` | 候选连接 | 4 道关卡检验框架 + 共识标签 + 输出清单 | 杀掉弱连接，保留真洞察 |

### 调用顺序

```
signal_decode → [验证] → insight_engine → [LLM 生成 5-6 候选] → reality_check → [LLM 筛选 2-3 存活] → 输出
```

## Slide 10 — Worked Examples

### Example 1: "最近好多人感冒了"

🔗 **生病的人不买药，他们买的是"无聊的解药"**

> 听到感冒多了，第一反应是药企。但药企在每年流感季几乎没有超额表现——流感是年年发生的事，早就在华尔街的模型里了。更有意思的方向是：大量请假在家的人不是在躺着养病，是在刷手机。
>
> 📚 2017-18 年 CDC 连续 19 周高于基线，Netflix 当季新增订户大幅超出预期，股价 +24%。
> ⚠️ 今天有 7-8 个平台竞争 + 短视频分流，传导效率可能已大幅下降。

🔗 **谁来顶你的班？——被忽视的二阶效应**

> 大规模缺勤→排班系统被挤压→企业被迫找临时工补位。效应滞后 2-3 周。
>
> 📚 2018 年缺勤率 3.0%，RHI 当季临时用工收入 +7%。
> ⚠️ 需要缺勤率 >3% 且宏观不在裁员周期。

💬 药企这个"显而易见"的角度我故意没展开——因为人人都想得到的，通常已经没有信息增量。你觉得哪条逻辑有问题？

### Example 2: "周一了，情绪低迷"（低信号 → 诚实回应）

🔗 **Monday Effect：学术界研究了 100 年的"周一诅咒"——然后它就消失了**

> 📚 1928-1982 年 S&P 500 周一平均回报 -0.13%（显著）。2003 年至今 +0.01%（不显著）。

💬 说实话，"周一情绪低迷"在金融层面信息量很低。如果你今天没有具体观察，最好的操作可能就是不操作。

🔗 **争议不是风险，争议是注意力货币——但这次变量不同**

> 📚 Nike-Kaepernick 2018：短期 -3.2%，90 天后 +18%。但 Nike 卖鞋，TSLA 卖车——决策链完全不同。

💬 这类名人争议事件的可预测性极低。如果你不持有相关标的，这件事大概率跟你没关系。

| 维度 | Mr.IF v4.3 | Mr.IF Spark v1.1 |

## Slide 13 — 产品哲学

### 三条核心信念

**1. 诚实 > 聪明**
> 我们宁可说"这件事我看不出特别有价值的金融角度"，也不愿制造看似聪明但经不住推敲的洞察。Beardstown Ladies 的教训是：看起来的聪明可以自我欺骗很久。

**2. 角度 > 答案**
> 我们不告诉你买什么。我们帮你想到你没想到的方向。方向可能对，可能错，但如果没有 Spark，你可能根本不会往那个方向想。价值在于"思考的起点"，不在于"结论的正确"。

**3. 弱点是特性，不是 Bug**
> 每条连接都标注⚠️最脆弱环节。每次输出都邀请用户反驳。我们不是在展示"我多聪明"，我们是在说"这是我的想法，来攻击它"。

### 一个类比

> Mr.IF Spark 就像一个读了很多书、对金融有独特直觉的朋友。
> 你跟 ta 聊天，说"最近好多人感冒了"，ta 不会说"多喝热水"，
> ta 会说"你知道吗，上次流感那么严重的时候，Netflix 的用户增长其实比药企的股价反应更大"。
> 然后你说"真的假的？"，ta 说"有可能我记错了，你去查查看"。
> 然后你去查了。然后你发现了一些有意思的东西。
> 那个"然后你自己去查了"的动作，就是 Spark 最大的价值。

## Slide 14 — 风险与局限性

### 我们诚实面对的问题

| 风险 | 描述 | 缓解措施 |
|------|------|---------|
| **幻觉风险** | LLM 可能编造不存在的历史数据 | Step 0 事实验证 + Step 2.5 网络检索补充 + 每条标注置信度来源 |
| **相关≠因果** | 蝴蝶效应链本质上是相关性推理 | Gate 1 因果测试 4 项检查 + Gate 3 做空者攻击 |
| **历史不代表未来** | 2017 年的 Netflix 逻辑在 2026 年未必成立 | 强制反例搜索 + ⚠️标注弱点 |
| **用户误用** | 用户可能把"思维练习"当成投资建议 | 明确说"不是投资建议" + 不给概率 + 不说"买" |
| **合规风险** | 涉及金融内容可能触碰监管红线 | 只到板块/主题，不推荐具体标的，每次输出含 disclaimer |
| **Beardstown Ladies 风险** | 系统可能产生"看似深刻但实际空洞"的输出 | 4 道 Gate + 模式单调检查 + 可以说"没什么" |
